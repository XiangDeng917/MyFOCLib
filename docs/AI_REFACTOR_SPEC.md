# AI 重构约束与交付规范（C 语言嵌入式项目）

## 1. 目标
本规范用于约束 AI 在本仓库执行重构、增量开发、文档生成时的行为，确保以下目标：

1. 代码风格统一且可读性高。
2. API 设计稳定、可复用、可移植。
3. 文档完整、标准化、可直接交付。
4. 改动可验证、可回归、可维护。

## 2. 强制执行项
AI 在任何重构任务中必须满足：

1. 不改变外部行为，除非明确说明并获得确认。
2. 优先小步重构，保持每一步可编译、可测试。
3. 新增公共接口必须写头文件注释和使用示例。
4. 每次重构必须给出风险点和验证方法。
5. 每次重构完成后必须提供变更清单与影响范围。

## 3. 命名与代码风格

### 3.1 命名规则
1. 函数名使用 `snake_case`，例如 `my_func`。
2. `PID` 缩写保持全大写，例如 `PID_controller_update`。
3. 结构体类型使用 `PascalCase`，例如 `PIDController`。
4. 结构体字段使用 `snake_case`，例如 `output_limit`。
5. 宏常量使用全大写下划线，例如 `MAX_OUTPUT_VOLTAGE`。
6. 文件名使用 `snake_case`，例如 `pid_controller.c`。

### 3.2 代码格式
1. 缩进使用 Tab 或项目既有风格，禁止混乱切换。
2. 每个函数控制在可读范围内，建议不超过 80 行。
3. 复杂逻辑前允许少量注释，注释解释“为什么”，不解释“显而易见的做了什么”。
4. 禁止魔法数字，必须提取为常量或配置项。
5. 头文件必须有 include guard。

### 3.3 设计约束
1. 核心算法层不得依赖具体芯片寄存器。
2. 平台相关代码放在 `port/` 或 `hal/` 目录。
3. 禁止跨层级直接引用（例如算法层直接 include 板级驱动头文件）。
4. 尽量消除全局可变状态，改用上下文句柄（handle）传递状态。

## 4. 可复用性与可移植性要求

### 4.1 接口设计
1. API 优先使用显式输入输出，不依赖隐式全局变量。
2. 支持多实例，避免单例状态污染。
3. 对时间源、IO、日志等外部能力采用依赖注入。
4. 配置与状态分离：`Config` 只读、`State` 可变。

### 4.2 通用 PID 要求
1. 标准更新接口应支持：`setpoint`、`measurement`、`Ts`。
2. 允许误差模式接口：`error`、`Ts`。
3. 必须支持输出限幅、积分限幅、输出斜率限制。
4. 默认参数必须安全，不允许未初始化直接输出异常值。

### 4.3 兼容策略
1. 重构时优先保留旧 API 兼容封装层。
2. 对废弃接口标记 `deprecated` 注释并给出迁移路径。
3. 重大变更必须提供迁移示例。

## 5. 文档交付标准
每次 AI 完成重构后，至少更新或产出以下文档：

1. `README.md`：模块简介、快速开始、构建方式、目录说明。
2. `API_REFERENCE.md`：每个公开函数的参数、返回值、边界条件、错误处理。
3. `USAGE.md`：最小可运行示例、典型场景示例、常见错误示例。
4. `MIGRATION.md`：旧 API 到新 API 的映射与替换示例。
5. `CHANGELOG.md`：按版本记录新增、变更、修复、破坏性更新。

## 6. AI 输出格式要求
AI 在完成代码任务后，回复必须包含：

1. 变更概述：改了什么、为什么改。
2. 文件级清单：每个文件的主要改动。
3. 风险与兼容性：可能影响与规避方法。
4. 验证结果：编译、静态检查、单元测试结果。
5. 后续建议：可选优化项，不强制执行。

## 7. 重构工作流（AI 执行步骤）
1. 阅读上下文并识别耦合点。
2. 先定义重构边界与不变行为。
3. 先抽象接口，再迁移实现。
4. 增加兼容层并保证旧调用可运行。
5. 增加示例与文档。
6. 执行验证并输出结果。

## 8. 验收清单
提交前逐项自检：

1. 是否符合命名风格（`snake_case`，`PID` 全大写）。
2. 是否移除或隔离硬件耦合。
3. 是否支持多实例与依赖注入。
4. 是否补齐文档与示例。
5. 是否保留兼容层或提供迁移文档。
6. 是否完成基本验证并记录结果。

## 9. 可直接复用的文档模板（建议）

### 9.1 API 注释模板
```c
/**
 * @brief 计算 PID 输出
 * @param pid PID 控制器句柄
 * @param setpoint 目标值
 * @param measurement 测量值
 * @param Ts 采样周期，单位秒
 * @return 控制输出
 * @note 输出受 output limits 与 ramp limit 约束
 */
float PID_controller_update(PIDController *pid, float setpoint, float measurement, float Ts);
```

### 9.2 使用示例模板
```c
PIDController pid;
PID_controller_init(&pid, 0.5f, 1.2f, 0.02f);
PID_controller_set_output_limits(&pid, -12.0f, 12.0f);
PID_controller_set_integral_limits(&pid, -6.0f, 6.0f);
PID_controller_set_output_ramp(&pid, 100.0f);

float u = PID_controller_update(&pid, target, feedback, 0.001f);
```

## 10. 禁止项
1. 未说明原因直接删除旧接口。
2. 将平台相关代码混入通用算法层。
3. 无验证结果直接声称重构完成。
4. 文档缺失或只给出片段化说明。
5. 引入新全局变量代替结构化状态管理。
